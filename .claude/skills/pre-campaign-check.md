# 告示前リスク詳細チェックスキル

下書きの公職選挙法リスクを詳しく分析します。自動チェック（hook）よりも深い判断を行います。

## 使い方

```
/pre-campaign-check drafts/instagram/YYYYMMDD_トピック.md
```

または全下書きを一括チェック:

```
/pre-campaign-check
```

---

## Claudeへの指示

このスキルが呼び出されたら、以下の手順を実行してください。

### Step 1: ファイルを読み込む

1. 引数で指定されたファイルを Read ツールで読み込む
2. 引数がない場合は `drafts/` 以下の全 `.md` ファイルをチェック対象とする
3. `checks/forbidden_words.md` を読み込む

### Step 2: 現在の選挙モードを確認

今日の日付（currentDate: 2026年3月29日）から現在のモードを確認:

- 〜2026/4/11: **事前運動禁止期間**（厳格なチェック）
- 2026/4/12〜4/18: **選挙運動期間**（投票呼びかけOK）
- 2026/4/19〜: **通常活動報告期間**

### Step 3: 詳細チェックを実施する

以下の観点で1文ずつ丁寧に分析する。

#### 3-1. CATEGORY_A 絶対禁止語チェック
- 告示前（現在）ならERROR
- 告示後ならINFO（合法）

#### 3-2. CATEGORY_B 要注意語チェック
単語だけでなく、**文脈も読んで判断**する:
- その文章全体として「投票・支持を求めているか？」
- 選挙の時期が近いことを意識させる内容か？
- 「普段の活動報告」と「選挙運動」のどちらに見えるか？

#### 3-3. 投稿全体のトーン評価
- この投稿を見た第三者は「事前運動」と受け取るか？
- 活動報告の文脈として自然か？
- 特定の政策への訴えが強すぎないか？

#### 3-4. 名前表記チェック
- 正しい表記が使われているか
- 誤表記がないか

#### 3-5. フロントマターの確認
- `status` が `posted` になっていないか（Claudeがセットしてはいけない）
- `date` が正しいか

### Step 4: リスク評価を出力する

以下のフォーマットで詳細なレポートを出力する:

```
## 公選法リスクチェックレポート
ファイル: drafts/xxx/YYYYMMDD_ttt.md
チェック日時: YYYY-MM-DD
期間モード: 事前運動禁止期間 / 選挙運動期間 / 通常期間

### 総合リスク評価: 🟢 低 / 🟡 中 / 🔴 高

**判断根拠**:
（1〜3行で総合評価の理由）

### 詳細所見

#### 🔴 ERROR（必ず修正）
（問題があれば記載。なければ「なし」）

#### ⚠️ WARNING（要確認）
（懸念があれば記載。なければ「なし」）

#### ✅ 問題なし
（問題のない点を列挙）

### 修正提案
（ERRORやWARNINGがある場合、具体的な修正案を提示）
```

### Step 5: 安全メモを更新する

チェック結果をもとに、ファイル内の `安全メモ` セクションを更新する（より詳細な情報を追記）。

## 判断の基準

### 「普段の活動報告」として許容されるもの
- 議会活動の報告（委員会・本会議・視察など）
- 地域イベントへの参加報告
- 政策課題の説明・現状報告
- 市民の声の紹介（個人情報なし）
- 季節・地域の話題

### 「事前運動」と判断されるリスクが高いもの
- 選挙を意識させる言葉（選挙・候補者・当選など）
- 有権者に何かを求める表現（投票・支持・応援など）
- 「これからも頑張る」の連呼（選挙運動的なアピール）
- 政治的訴えの過剰な強調

---

## 最終注意

> このスキルの判断はClaude（AI）による推定です。
> 公職選挙法の法的判断は最終的に人間（本人または弁護士）が行ってください。
> 迷う場合は使わないことが最も安全な選択肢です。
