# 実運用手順書

こぼり良江 SNS運用支援システムの完全な運用マニュアルです。

---

## 1. システム概要

### 目的

このシステムは、栃木市議会議員・こぼり良江（小堀良江）さんのSNS活動を補助します。

- 1つの活動メモから Instagram / X / Facebook の下書きを生成
- 公職選挙法に配慮した自動チェック
- 必ず人間がレビューしてから手動投稿（自動投稿なし）

### システム構成図

```
あなた（活動後）
    ↓
ideas/ に活動メモを作成
    ↓
Claude Code で /instagram-draft 等を実行
    ↓
drafts/ に下書きが保存される
    ↓ （hookが自動チェック実行）
チェック結果を確認
    ↓
あなたがレビュー・必要に応じ修正
    ↓
status: approved に変更
    ↓
あなたがSNSに手動投稿
    ↓
status: posted に変更・投稿日時を記録
```

## 3. 活動メモの書き方

### テンプレートの使い方

```bash
cp ideas/TEMPLATE_idea.md ideas/YYYYMMDD_トピック.md
```

### 各セクションの書き方

**`## 活動メモ`**: 事実ベースで記述。5W1H（いつ・どこで・誰と・何を・なぜ・どう）を意識する。

**`## 伝えたいこと`**: 市民に届けたい核心メッセージ。1〜3つに絞る。これがないと良い投稿文が生まれない。

**`## 使える素材`**: 写真の有無・使えるデータを記録しておくと、後で下書き生成がスムーズ。

### 良い活動メモの例
```
## 活動メモ
3月28日、市内の保育所を視察。園長から施設老朽化と処遇改善の課題を聞いた。
保護者3名と意見交換し、延長保育拡充の要望を多数いただいた。

## 伝えたいこと
- 現場の実態を知ったこと
- 保育士の処遇改善が喫緊の課題
- 議会で取り上げる意志
```

### 避けるべき活動メモの書き方
- 抽象的すぎる（「いろいろ視察しました」）
- 個人情報が含まれる（氏名・住所・電話番号）
- 私的な内容（SNS投稿には不適切なもの）

## 4. 下書きレビューチェックリスト

以下を確認してから `status: approved` にしてください。

### 基本確認
- [ ] チェック結果に `❌ ERROR` がないか
- [ ] チェック結果の `⚠️ WARNING` を確認し、問題ないと判断したか
- [ ] 名前表記が `こぼり良江` または `小堀良江` になっているか

### 内容確認
- [ ] 事実の記述に誤りがないか（日付・場所・人物・数字）
- [ ] 文体がそのプラットフォームに合っているか
- [ ] 個人情報が含まれていないか（特定の個人の氏名・住所等）

### 公選法確認（告示前）
- [ ] 安全メモを確認した
- [ ] 投票依頼・支持依頼の表現がないか
- [ ] 全体のトーンが「事前運動」に見えないか
- [ ] 特に心配な場合: `/pre-campaign-check` を実施した

### 媒体別確認
- [ ] **Instagram**: 画像は準備できているか。ハッシュタグは15個以上か
- [ ] **X**: 文字数は140文字以内か。URLを含む場合は23文字換算で確認したか
- [ ] **Facebook**: 公開範囲は適切か（全体公開 / 友達 / フォロワー）

## 5. プラットフォーム別投稿戦略

### Instagram
**特性**: 写真・ビジュアルが重要。ハッシュタグで新規フォロワーを獲得しやすい。

**推奨頻度**: 週2〜3回

**投稿に向いている内容**:
- 現地視察・イベント参加（写真あり）
- 地域の風景・季節感を絡めた活動報告
- 子育て・福祉など身近なテーマ

**タイミング**: 午前7〜9時、または正午（エンゲージメントが高い）

---

### X（Twitter）
**特性**: リアルタイム性・拡散性が高い。速報・短い報告に向いている。

**推奨頻度**: 週3〜5回（イベント・議会時は当日投稿）

**投稿に向いている内容**:
- 議会・委員会の報告（当日・翌日）
- 市政の動向・数字のある報告
- 時事に絡めたコメント

**タイミング**: 午前8時、または夕方17〜19時

---

### Facebook
**特性**: 地元の知人・支持者が多い。長文OKで、丁寧な説明ができる。

**推奨頻度**: 週1〜2回

**投稿に向いている内容**:
- 政策説明・課題の背景を丁寧に解説
- 議会報告（週次まとめ）
- 地域の重要なお知らせ

**タイミング**: 午前10時、または夕方18〜20時

## 6. 公職選挙法 運用カレンダー

### 現在（〜2026年4月11日）: 事前運動禁止期間

このシステムのデフォルト設定です。

**OK**: 普段の議員活動報告・政策説明・地域情報
**NG**: 投票依頼・支持依頼・選挙を意識させる表現

### 選挙運動期間（2026年4月12日〜18日）

告示日（4月12日）以降は選挙運動として積極的な発信が可能です。

**OK（追加で可能になること）**:
- 「ぜひ投票をお願いします」
- 「こぼり良江への一票をよろしくお願いします」
- 選挙・候補者としての訴え

**check_draft.shは自動的にモードを切り替えます**（日付で判定）。

### 投開票日（2026年4月19日）

投稿は停止してください。

### 投開票後（2026年4月20日〜）

通常の議員活動報告モードに戻ります。

## 7. 禁止語・要注意語の管理

`checks/forbidden_words.md` に記載されています。

### 新しい表現を追加したい場合

ファイルを直接編集してください:

```markdown
## CATEGORY_A: 絶対禁止
（新しい禁止語を追加）

## CATEGORY_B: 要注意
（新しい要注意語を追加）
```

check_draft.sh はこのファイルとは独立して動作するため、
スクリプト側の配列（FORBIDDEN_A・CAUTION_B）も更新が必要です。
変更が必要な場合は Claude Code に依頼してください。

## 8. トラブルシューティング

| 問題 | 確認すること | 対処 |
|------|------------|------|
| hookが動かない | `chmod +x .claude/hooks/check_draft.sh` | 実行権限を付与 |
| チェック結果が更新されない | ファイルがdrafts/以下か確認 | パスを確認してから再保存 |
| X投稿が140文字超 | char_countを確認 | スレッド形式に分割を依頼 |
| ハッシュタグ数の警告 | instagram: 15個以上、x: 3個以下 | hashtag_bank.mdを参照して調整 |
| 名前表記エラー | こぼり良江/小堀良江以外が含まれている | 該当箇所を修正 |
| jqがない | `which jq` で確認 | `sudo apt install jq` |

## 9. 定期メンテナンス

### 月次
- `drafts/` のposted済みファイルを `archive/` に移動（任意）
- 禁止語リストの見直し（新しいリスクワードの追加）

### 選挙期間後
- `check_draft.sh` の日付定数を次の選挙に合わせて更新
  - `ELECTION_START`・`ELECTION_END`・`ELECTION_DAY` の値を変更

- ハッシュタグバンクの見直し（人気タグの更新）
- スキルファイルの見直し（文体ガイドの更新）
