## プラットフォーム別文体ガイド

### Instagram（@kobori_yoshie）
- **トーン**: 親しみやすく、温かく、市民目線の体験談・感想
- **文体**: 体言止め・感嘆表現OK・絵文字を適度に使う
- **本文**: 300〜800文字（長くなりすぎず読みやすく）
- **ハッシュタグ**: 15〜25個（最初の3つは固定）
- **改行**: こまめに入れてモバイルで読みやすく

### X / Twitter（@kobori_yoshie）
- **トーン**: やや硬め・簡潔・事実ベース
- **文体**: 敬語・報告調・絵文字は最小限
- **本文**: **140文字以内**（ハッシュタグ含む、URLは23文字換算）
- **ハッシュタグ**: 1〜3個（文末に置く）
- **スレッド**: 内容が長い場合は1/n形式で複数投稿案を作る

### Facebook
- **トーン**: 信頼感・丁寧語・地域リーダーとしての言葉
- **文体**: 敬語・ですます調・政策背景の説明もOK
- **本文**: 200〜500文字（フレキシブル）
- **ハッシュタグ**: 0〜5個（少なめが効果的）
- **フォーマット**: 段落分けしてPC閲覧でも読みやすく

## 公職選挙法 安全ルール

### 告示前（〜2026/4/11）に絶対禁止の表現
```
投票をお願い / 投票してください / ぜひ投票を
支持をお願い / 支持してください
当選させてください
○○に一票 / 一票をお願い
```

### 告示前に要注意（文脈次第でNG）
```
応援してください（選挙文脈に見える場合）
拡散をお願いします（集票目的に見える場合）
皆さまのお力添えを
選挙に向けて頑張ります
```

### 告示後（2026/4/12〜4/18）はOK
告示後の選挙運動期間中は、投票を呼びかける文言を使用できます。
ただし **告示前の投稿に混入しないよう**、日付に注意してください。

### 迷ったときのルール
> 「この表現、投票を求めているように見えるかも？」と思ったら使わない。
> `安全メモ` に記録して、人間がレビューすること。

## スキル一覧（使い方）

| コマンド | 機能 |
|---------|------|
| `/instagram-draft ideas/ファイル.md` | Instagram下書きを生成 |
| `/x-draft ideas/ファイル.md` | X下書きを生成 |
| `/facebook-draft ideas/ファイル.md` | Facebook下書きを生成 |
| `/weekly-summary` | 今週のideasから投稿カレンダーを作成 |
| `/pre-campaign-check drafts/プラットフォーム/ファイル.md` | 告示前リスクを詳細チェック |

## ワークフロー

```
1. 活動後すぐ
   ideas/YYYYMMDD_トピック.md を作成（TEMPLATE_idea.md をコピー）

2. 下書き生成
   /instagram-draft ideas/ファイル.md
   /x-draft ideas/ファイル.md
   /facebook-draft ideas/ファイル.md

3. 自動チェック（保存時に自動実行）
   → 禁止語・文字数・ハッシュタグ数・表記ゆれを自動チェック
   → 結果は各下書きの「チェック結果」セクションに記録

4. 人間レビュー
   drafts/プラットフォーム/ファイル.md を確認
   問題があれば修正・/pre-campaign-check で詳細確認

5. 投稿（手動）
   承認したら status: approved に変更
   各SNSに手動でコピー&ペーストして投稿

6. 記録
   投稿後に status: posted と posted_at: 日時 を記入
```

## Claudeが絶対にやってはいけないこと

- SNSへの自動投稿・API連携
- `status: posted` を自分でセットする（人間が投稿後に手動で行う）
- 安全メモを省略する
- 告示前に投票依頼・支持依頼の表現を生成する
- 名前の誤表記（チェックを通過した場合でも自覚的に修正する）
